## Sección Técnica

### Estrategia de Implementación

1. **Verificar items existentes**: Consultar la base de datos para ver qué items ya existen y evitar duplicados

2. **Insertar items nuevos**: Usar SQL INSERT para agregar todos los items del menú a la tabla `menu_items`

3. **Estructura de datos por item**:
```text
- category: starters | mains | desserts | drinks
- subcategory: null (para comida) | cocktails | low_alcohol | red_wine | white_wine | rose_wine | sparkling_wine
- name_es: nombre en español (minúsculas)
- name_en: nombre en inglés (a definir después o dejar igual por ahora)
- description_es: descripción en español
- description_en: descripción en inglés (a definir después)
- price: formato "₡X,XXX" o "copa ₡X,XXX · botella ₡XX,XXX" para vinos
- is_available: true
- sort_order: número secuencial por categoría
```

4. **Para los nombres en inglés**: Usaré el nombre en español como placeholder temporal, ya que el documento no incluye las traducciones al inglés

### Notas del Documento a Respetar
- Todo en minúscula (excepto DOC, DO, DOCa)
- [v] indica vegetariano
- Los vinos tienen formato especial con copa/botella
